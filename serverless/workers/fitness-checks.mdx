Fitness checks verify that your <WorkersTooltip /> are ready to process jobs before accepting requests. The Runpod SDK runs these checks once at startup, confirming that system resources, GPU availability, and any custom requirements are met before your worker begins handling workloads.

If any check fails, the worker exits immediately and is marked as unhealthy. This prevents unhealthy workers from processing jobs and causing failures.

## Custom fitness checks

Register custom fitness checks using the `@runpod.serverless.register_fitness_check` decorator. Custom checks run before built-in system checks, letting you validate application-specific requirements like model files, environment variables, or external service connectivity.

```python handler.py
import runpod

@runpod.serverless.register_fitness_check
def check_model_files():
    """Verify required model files exist."""
    from pathlib import Path
    model_path = Path("/models/my-model.safetensors")
    if not model_path.exists():
        raise RuntimeError(f"Model not found: {model_path}")

@runpod.serverless.register_fitness_check
async def check_external_api():
    """Verify external API is accessible."""
    import aiohttp
    async with aiohttp.ClientSession() as session:
        async with session.get("https://api.example.com/health", timeout=5) as resp:
            if resp.status != 200:
                raise RuntimeError(f"API health check failed: {resp.status}")

def handler(job):
    return {"output": "success"}

runpod.serverless.start({"handler": handler})
```

Key points for custom checks:

- Both sync and async functions are supported.
- Raise an exception to signal failure. No return value is needed for success.
- Add checks anywhere in your code before calling `runpod.serverless.start()`.
- Checks run in the order they are registered.

### Common check patterns

**Verify environment variables:**

```python
@runpod.serverless.register_fitness_check
def check_required_env_vars():
    """Ensure required environment variables are set."""
    import os
    required = ["MODEL_NAME", "API_KEY", "CACHE_DIR"]
    missing = [var for var in required if not os.getenv(var)]
    if missing:
        raise RuntimeError(f"Missing environment variables: {', '.join(missing)}")
```

**Validate GPU availability:**

```python
@runpod.serverless.register_fitness_check
def check_gpu_memory():
    """Verify GPU has sufficient memory for the model."""
    import torch
    if not torch.cuda.is_available():
        raise RuntimeError("CUDA not available")

    gpu_memory_gb = torch.cuda.get_device_properties(0).total_memory / 1e9
    required_memory_gb = 16  # Adjust based on your model
    if gpu_memory_gb < required_memory_gb:
        raise RuntimeError(f"Insufficient GPU memory: {gpu_memory_gb:.1f}GB (need {required_memory_gb}GB)")
```

**Check external service connectivity:**

```python
@runpod.serverless.register_fitness_check
async def check_database_connection():
    """Verify database is accessible."""
    import asyncpg
    import os
    try:
        conn = await asyncpg.connect(os.getenv("DATABASE_URL"), timeout=5)
        await conn.close()
    except Exception as e:
        raise RuntimeError(f"Database connection failed: {e}")
```

## Built-in system checks

The SDK includes built-in checks that validate system resources and GPU health. These run automatically after any custom checks you define.

### All workers

Every worker runs these three checks:

| Check | Description | Default | Environment variable |
|-------|-------------|---------|---------------------|
| Memory | Validates available RAM meets minimum threshold | 4 GB | `RUNPOD_MIN_MEMORY_GB` |
| Disk space | Validates free disk space as a percentage of total | 10% | `RUNPOD_MIN_DISK_PERCENT` |
| Network | Validates internet connectivity | 5s timeout | `RUNPOD_NETWORK_CHECK_TIMEOUT` |

### GPU workers

GPU workers run four additional checks:

| Check | Description | Default | Environment variable |
|-------|-------------|---------|---------------------|
| CUDA version | Validates CUDA driver version meets minimum | 11.8 | `RUNPOD_MIN_CUDA_VERSION` |
| CUDA initialization | Verifies GPU devices can be initialized and memory allocated | N/A | N/A |
| GPU compute benchmark | Runs a quick matrix multiplication to validate performance | 100ms timeout | `RUNPOD_GPU_BENCHMARK_TIMEOUT` |
| GPU binary test | Comprehensive GPU health check via native binary | N/A | N/A |

<Note>
GPU checks are skipped automatically on CPU-only workers. The same code works for both CPU and GPU deployments without modification.
</Note>

## Configuration

Configure built-in check thresholds using environment variables in your Dockerfile or at runtime:

```dockerfile Dockerfile
ENV RUNPOD_MIN_MEMORY_GB=8.0
ENV RUNPOD_MIN_DISK_PERCENT=15.0
ENV RUNPOD_MIN_CUDA_VERSION=12.0
ENV RUNPOD_NETWORK_CHECK_TIMEOUT=10
ENV RUNPOD_GPU_BENCHMARK_TIMEOUT=2
```

Or set them in Python before starting your handler:

```python handler.py
import os
os.environ["RUNPOD_MIN_MEMORY_GB"] = "8.0"
os.environ["RUNPOD_MIN_DISK_PERCENT"] = "15.0"
```

### GPU binary test configuration

The GPU binary test has additional configuration options:

| Environment variable | Description | Default |
|---------------------|-------------|---------|
| `RUNPOD_GPU_TEST_TIMEOUT` | Timeout in seconds for the GPU test | 30 |
| `RUNPOD_GPU_MAX_ERROR_MESSAGES` | Maximum number of error messages parsed from GPU test output | 10 |
| `RUNPOD_BINARY_GPU_TEST_PATH` | Custom path to the GPU test binary | System default |

## Disabling built-in checks

For testing or specialized deployments, you can disable built-in checks using environment variables. This is not recommended for production use.

| Environment variable | Effect |
|---------------------|--------|
| `RUNPOD_SKIP_AUTO_SYSTEM_CHECKS=true` | Skips auto-registration of memory, disk, network, CUDA version, CUDA init, and GPU benchmark checks |
| `RUNPOD_SKIP_GPU_CHECK=true` | Skips auto-registration of the GPU binary test |

```python handler.py
import os

# Disable all auto-registered system checks (testing only)
os.environ["RUNPOD_SKIP_AUTO_SYSTEM_CHECKS"] = "true"

# Disable the GPU binary test
os.environ["RUNPOD_SKIP_GPU_CHECK"] = "true"
```

<Warning>
Custom fitness checks registered with `@runpod.serverless.register_fitness_check` still run even when these skip flags are set.
</Warning>


## Failure behavior

When any fitness check fails:

1. The worker logs the failure with details about what went wrong.
2. The worker exits immediately with exit code 1.
3. Runpod marks the worker as unhealthy.
4. The system attempts to start a replacement worker.

This fail-fast approach prevents partially functional workers from accepting jobs and producing errors or unexpected results.
