List available GPU types and their specifications.

<RequestExample>
```bash Command
runpodctl gpu <subcommand> [flags]
```
</RequestExample>

## Subcommands

### List available GPUs

List GPUs that are currently available:

```bash
runpodctl gpu list
```

Include unavailable GPUs in the list:

```bash
runpodctl gpu list --include-unavailable
```

#### List flags

<ResponseField name="--include-unavailable" type="bool">
Include GPUs that are currently unavailable.
</ResponseField>

## Output fields

Each GPU type in the output includes the following fields:

<ResponseField name="gpuId" type="string">
The GPU type ID to pass to `--gpu-id` when creating Pods or endpoints.
</ResponseField>

<ResponseField name="displayName" type="string">
A human-friendly name for the GPU type.
</ResponseField>

<ResponseField name="memoryInGb" type="int">
GPU memory in gigabytes.
</ResponseField>

<ResponseField name="secureCloud" type="bool">
Whether the GPU type is offered on Secure Cloud.
</ResponseField>

<ResponseField name="communityCloud" type="bool">
Whether the GPU type is offered on Community Cloud.
</ResponseField>

<ResponseField name="securePricePerHr" type="float">
On-demand Secure Cloud price in USD per hour. Omitted when Secure Cloud does not offer the GPU type.
</ResponseField>

<ResponseField name="communityPricePerHr" type="float">
On-demand Community Cloud price in USD per hour. Omitted when Community Cloud does not offer the GPU type.
</ResponseField>

<ResponseField name="stockStatus" type="string">
The best availability across all datacenters: `High`, `Medium`, or `Low`.
</ResponseField>

<ResponseField name="dataCenterAvailability" type="array">
A per-datacenter availability breakdown. Each entry contains a `dataCenterId` and its `stockStatus` (`High`, `Medium`, `Low`, or `none` when the GPU type is offered in that datacenter but currently has no stock).
</ResponseField>

<ResponseField name="available" type="bool">
Whether the GPU type currently has availability.
</ResponseField>

Use the `securePricePerHr` and `communityPricePerHr` fields to compare on-demand costs across cloud types. Use `dataCenterAvailability` to pick a datacenter that can actually schedule your workload, rather than relying on the best-overall `stockStatus`.

## Using GPU IDs

When creating Pods or Serverless endpoints, use the GPU ID from the list with the `--gpu-id` flag:

```bash
runpodctl pod create --template-id runpod-torch-v21 --gpu-id "NVIDIA RTX 4090"
```
