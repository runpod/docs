Manage Pods, including creating, listing, starting, stopping, and deleting Pods.

<RequestExample>
```bash Command
runpodctl pod <subcommand> [flags]
```
</RequestExample>

## Subcommands

### List Pods

List all your Pods:

```bash
runpodctl pod list
```

### Get Pod details

Get detailed information about a specific Pod, including SSH connection info:

```bash
runpodctl pod get <pod-id>
```

### Create a Pod

Create a new Pod from a template:

```bash
runpodctl pod create --template-id runpod-torch-v21 --gpu-id "NVIDIA RTX 4090"
```

Create a Pod with a custom Docker image:

```bash
runpodctl pod create --image "runpod/pytorch:2.1.0-py3.10-cuda11.8.0-devel-ubuntu22.04" --gpu-id "NVIDIA RTX 4090"
```

Create a CPU-only Pod:

```bash
runpodctl pod create --compute-type cpu --image ubuntu:22.04
```

#### Create flags

| Flag | Type | Description |
|------|------|-------------|
| `--template-id` | string | Template ID to use for Pod configuration |
| `--image` | string | Docker image to use (e.g., `runpod/pytorch:latest`) |
| `--name` | string | Custom name for the Pod |
| `--gpu-id` | string | GPU type (e.g., `NVIDIA RTX 4090`, `NVIDIA A100 80GB PCIe`) |
| `--gpu-count` | int | Number of GPUs to allocate |
| `--compute-type` | string | Compute type (`gpu` or `cpu`) |
| `--container-disk-in-gb` | int | Container disk size in GB |
| `--volume-in-gb` | int | Persistent volume size in GB |
| `--volume-mount-path` | string | Mount path for the persistent volume |
| `--network-volume-id` | string | ID of a network volume to attach |
| `--ports` | string | Ports to expose (e.g., `8888/http`) |
| `--env` | string | Environment variables (can be specified multiple times) |
| `--cloud-type` | string | Cloud tier (`SECURE` or `COMMUNITY`) |
| `--data-center-ids` | string | Preferred datacenter IDs |
| `--global-networking` | bool | Enable global networking |
| `--public-ip` | bool | Enable public IP |

### Start a Pod

Start a stopped Pod:

```bash
runpodctl pod start <pod-id>
```

### Stop a Pod

Stop a running Pod:

```bash
runpodctl pod stop <pod-id>
```

### Restart a Pod

Restart a Pod:

```bash
runpodctl pod restart <pod-id>
```

### Reset a Pod

Reset a Pod to its initial state:

```bash
runpodctl pod reset <pod-id>
```

### Update a Pod

Update Pod configuration:

```bash
runpodctl pod update <pod-id> --name "new-name"
```

### Delete a Pod

Delete a Pod:

```bash
runpodctl pod delete <pod-id>
```

## Pod URLs

Access exposed ports on your Pod using the following URL pattern:

```
https://<pod-id>-<port>.proxy.runpod.net
```

For example, if your Pod ID is `abc123xyz` and you exposed port 8888:
```
https://abc123xyz-8888.proxy.runpod.net
```
