Configure the Runpod CLI with your API credentials and API URL to enable programmatic access to your Runpod resources.

<RequestExample>
```bash Command
runpodctl config [flags]
```
</RequestExample>

## Example

Configure the CLI with your API key:

```bash
runpodctl config --apiKey "your-api-key-here"
```

<Note>

For first-time setup, we recommend using [`runpodctl doctor`](/runpodctl/reference/runpodctl-doctor) instead, which guides you through API key configuration and SSH key setup interactively.

</Note>

## Flags

<ResponseField name="--apiKey" type="string">
Your Runpod API key, which authenticates the CLI to access your account. You can generate an API key from the [Runpod console](https://www.runpod.io/console/user/settings).
</ResponseField>

<ResponseField name="--apiUrl" type="string" default="https://api.runpod.io/graphql">
The Runpod API endpoint URL. The default value should work for most users.
</ResponseField>

## Environment variables

The Runpod CLI also reads your API key from the `RUNPOD_API_KEY` environment variable. When this variable is set, it takes precedence over the `apiKey` value in your config file, so you can override your configured credentials for a single session or project without editing the file.

Setting the environment variable is a convenient alternative to `runpodctl config` when you want to keep your API key out of the config file, such as in CI environments or on shared machines:

```bash
export RUNPOD_API_KEY=your-api-key-here
```

The CLI uses the first credential it finds, in this order: the `RUNPOD_API_KEY` environment variable, then the `apiKey` value in your config file. If neither is set, commands that require authentication fail with an error prompting you to configure your API key.
