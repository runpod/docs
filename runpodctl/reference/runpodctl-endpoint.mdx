Manage Serverless endpoints.

```bash
runpodctl endpoint <subcommand> [flags]
```

## Subcommands

### Update model references

Set the model references (cached models) for a Serverless endpoint. Model references specify which models should be cached on your endpoint's workers, improving cold start times for those models.

```bash
runpodctl endpoint model <endpoint-id> <model-ref> [model-ref...]
```

**Example:**

```bash
runpodctl endpoint model abc123xyz meta-llama/Llama-2-7b-chat-hf
```

To set multiple model references:

```bash
runpodctl endpoint model abc123xyz runpod/stable-diffusion-v1-5 meta-llama/Llama-2-7b-chat-hf
```

#### Arguments

| Argument | Description |
|----------|-------------|
| `endpoint-id` | The ID of the endpoint to update. Use [`runpodctl serverless list`](/runpodctl/reference/runpodctl-serverless) to find your endpoint IDs. |
| `model-ref` | One or more model references. Format: `owner/model-name` or `owner/model-name:branch`. |
