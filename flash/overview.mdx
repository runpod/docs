Flash is a Python SDK for developing cloud-native AI apps where you define everything—hardware, remote functions, and dependencies—using local code.

Write `@remote` decorated Python functions on your local machine. Run them, and Flash automatically handles GPU/CPU provisioning and worker scaling on [Runpod Serverless](/serverless/overview).

## Setup

### Install Flash

<Note>
Flash requires [Python 3.10+](https://www.python.org/downloads/), and is currently available for macOS and Linux. Windows support is in development.
</Note>

Install Flash using `pip` or `uv`:

```bash
# Install with pip
pip install runpod-flash

# Or uv
uv add runpod-flash
```

### Authentication

Before you can use Flash, you need to authenticate with your Runpod account:

```bash
flash login
```

This saves your API key securely and allows you to use the Flash CLI and run `@remote` functions.

### Coding agent integration (optional)

Install the Flash skill package for AI coding agents like Claude Code, Cline, and Cursor:

```bash
npx skills add runpod/skills
```

You can review the Flash `SKILL.md` file in the [runpod/skills repository](https://github.com/runpod/skills/blob/main/flash/SKILL.md).

## Core concepts

### Resource configuration

Flash provides fine-grained control over hardware allocation through configuration objects. Choose from GPU or CPU configurations, and customize worker counts, scaling behavior, and more.

```python
from runpod_flash import LiveServerless, GpuGroup

gpu_config = LiveServerless(
    name="ml-inference",
    gpus=[GpuGroup.AMPERE_80],  # A100 80GB
    workersMax=5,
    idleTimeout=10
)
```

[Learn more about resource configuration](/flash/configuration/overview).

### Remote functions

The `@remote` decorator marks functions for execution on Runpod's infrastructure. Code inside the decorated function runs remotely on a Serverless worker, while code outside the function runs locally on your machine.

```python
# Runs remotely using the resource configuration above
@remote(resource_config=gpu_config, dependencies=["torch"])
async def get_gpu_name():
    import torch  # Import packages inside the function body
    device_name = torch.cuda.get_device_name(0)  # Get GPU name
    return {"gpu": device_name}

async def main():
    print("Running on Runpod...")  # Runs locally
    result = await get_gpu_name()  # Calls the remote function
    print(result["gpu"])  # Runs locally
```

When you call a remote function, Flash:
- Automatically provisions resources on Runpod's infrastructure.
- Installs your dependencies on the remote worker.
- Executes your function on a remote GPU/CPU.
- Returns the result to your local environment.

[Learn more about remote functions](/flash/remote-functions).

### Flash apps for production APIs

When you're ready to move beyond scripts and build a production-ready API, you can create a [Flash app](/flash/apps/overview) and deploy it to Runpod. Flash apps provide a complete development and deployment workflow with local testing and production deployment.

[Follow this tutorial to build your first Flash app](/flash/apps/build-app).

## Why use Flash?

- **No Docker images required:** Flash deploys infrastructure automatically when you run your code—no container builds, no UI steps.
- **Control everything from local code:** Define remote functions, specify exact hardware, and configure scaling, all from Python scripts that you write and run locally.
- **Instant updates:** Update your code and run it again—it deploys instantly without rebuilding the container image.
- **Pay only for compute time:** Flash uses the same per-second pricing model as [Runpod Serverless](/serverless/pricing).

## Limitations

- Flash is currently only available for macOS and Linux. Windows support is in development.
- Serverless deployments using Flash are currently restricted to the `EU-RO-1` datacenter.
- Be aware of your account's maximum worker capacity limits. Flash can rapidly scale workers across multiple endpoints, and you may hit capacity constraints. Contact [Runpod support](https://www.runpod.io/contact) to increase your account's capacity allocation if needed.
