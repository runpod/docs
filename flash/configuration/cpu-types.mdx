Flash provides access to CPU-only compute instances for workloads that don't require GPU acceleration. This reference lists all available CPU instance types and their specifications.

## CPU instance types

The `CpuInstanceType` enum provides access to CPU-only compute instances. Use these for workloads that don't require GPU acceleration.

### Available CPU instance types

CPU instances are organized by generation and optimization profile:

#### 5th generation compute-optimized

Latest generation, optimized for compute-intensive workloads:

| CpuInstanceType | ID | vCPU | RAM | Best For |
|-----------------|-----|------|-----|----------|
| `CPU5C_1_2` | cpu5c-1-2 | 1 | 2GB | Lightweight APIs, simple tasks |
| `CPU5C_2_4` | cpu5c-2-4 | 2 | 4GB | Small APIs, data validation |
| `CPU5C_4_8` | cpu5c-4-8 | 4 | 8GB | General APIs, data processing |
| `CPU5C_8_16` | cpu5c-8-16 | 8 | 16GB | Heavy processing, parallel tasks |

#### 3rd generation compute-optimized

Balanced compute focus:

| CpuInstanceType | ID | vCPU | RAM | Best For |
|-----------------|-----|------|-----|----------|
| `CPU3C_1_2` | cpu3c-1-2 | 1 | 2GB | Basic endpoints, webhooks |
| `CPU3C_2_4` | cpu3c-2-4 | 2 | 4GB | Simple data processing |
| `CPU3C_4_8` | cpu3c-4-8 | 4 | 8GB | Moderate workloads |
| `CPU3C_8_16` | cpu3c-8-16 | 8 | 16GB | CPU-intensive tasks |

#### 3rd generation general purpose

Balanced CPU and memory:

| CpuInstanceType | ID | vCPU | RAM | Best For |
|-----------------|-----|------|-----|----------|
| `CPU3G_1_4` | cpu3g-1-4 | 1 | 4GB | Memory-light tasks |
| `CPU3G_2_8` | cpu3g-2-8 | 2 | 8GB | General workloads |
| `CPU3G_4_16` | cpu3g-4-16 | 4 | 16GB | Memory-intensive processing |
| `CPU3G_8_32` | cpu3g-8-32 | 8 | 32GB | High-memory workloads |

### Using CPU instance types

Import and use CPU instance types in your configurations:

```python
from runpod_flash import CpuLiveServerless, CpuInstanceType

# Single instance type
config = CpuLiveServerless(
    name="data-processor",
    instanceIds=[CpuInstanceType.CPU5C_4_8]
)

# Multiple instance types (fallback strategy)
config = CpuLiveServerless(
    name="flexible-cpu",
    instanceIds=[
        CpuInstanceType.CPU5C_8_16,  # Try 5th gen, 8 vCPU first
        CpuInstanceType.CPU5C_4_8,   # Fall back to 4 vCPU
        CpuInstanceType.CPU3G_4_16   # Final fallback to 3rd gen
    ]
)
```

### Choosing CPU instance types

**For APIs and webhooks:**
```python
# Lightweight API
instanceIds=[CpuInstanceType.CPU5C_2_4]  # 2 vCPU, 4GB

# Production API
instanceIds=[CpuInstanceType.CPU5C_4_8]  # 4 vCPU, 8GB
```

**For data processing:**
```python
# Light processing
instanceIds=[CpuInstanceType.CPU3G_2_8]  # 2 vCPU, 8GB (more RAM)

# Heavy processing
instanceIds=[CpuInstanceType.CPU5C_8_16]  # 8 vCPU, 16GB
```

**For memory-intensive tasks:**
```python
# High memory requirement
instanceIds=[CpuInstanceType.CPU3G_8_32]  # 8 vCPU, 32GB RAM
```
